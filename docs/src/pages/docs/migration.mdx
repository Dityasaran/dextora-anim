# Migration from AOS

Switching from AOS (Animate On Scroll) to data-anim is straightforward. The attribute-based API is intentionally similar, so most changes are simple find-and-replace operations.

## Step 1: Replace the Script

Remove AOS CSS and JS. Add data-anim.

<CodeBlock
  code={'<!-- Remove these -->\n<link rel="stylesheet" href="https://unpkg.com/aos@2/dist/aos.css" />\n<script src="https://unpkg.com/aos@2/dist/aos.js"></script>\n<script>AOS.init();</script>\n\n<!-- Add this -->\n<script src="https://unpkg.com/data-anim@latest/dist/data-anim.js"></script>'}
  lang="html"
/>

## Step 2: Replace Attributes

The main change is renaming `data-aos` to `data-anim` and updating attribute names.

### Attribute Mapping

| AOS Attribute | data-anim Attribute | Notes |
|---------------|---------------------|-------|
| `data-aos="fade-up"` | `data-anim="fadeInUp"` | camelCase naming |
| `data-aos="fade-down"` | `data-anim="fadeInDown"` | |
| `data-aos="fade-left"` | `data-anim="fadeInLeft"` | |
| `data-aos="fade-right"` | `data-anim="fadeInRight"` | |
| `data-aos="slide-up"` | `data-anim="slideInUp"` | |
| `data-aos="zoom-in"` | `data-anim="zoomIn"` | |
| `data-aos="zoom-out"` | `data-anim="zoomOut"` | |
| `data-aos="flip-up"` | `data-anim="flip"` | |
| `data-aos-duration` | `data-anim-duration` | Same format |
| `data-aos-delay` | `data-anim-delay` | Same format |
| `data-aos-easing` | `data-anim-easing` | CSS easing values |
| `data-aos-once` | `data-anim-once` | Same values |
| `data-aos-offset` | `data-anim-offset` | Same values |
| `data-aos-mirror` | `data-anim-mirror` | Same values |

## Step 4: Remove FOUC Workarounds

If you added manual CSS to prevent AOS FOUC, you can remove it:

<CodeBlock
  code={'/* Remove all of this \u2014 data-anim handles it automatically */\n[data-aos] {\n  opacity: 0;\n  transition-property: opacity, transform;\n}\n[data-aos].aos-animate {\n  opacity: 1;\n}\nbody[data-aos-easing="ease"] [data-aos] {\n  transition-timing-function: ease;\n}'}
  lang="css"
/>

## Quick Find & Replace

For most projects, these regex replacements handle the bulk of the migration:

```
Find:    data-aos="fade-up"
Replace: data-anim="fadeInUp"

Find:    data-aos="fade-down"
Replace: data-anim="fadeInDown"

Find:    data-aos="fade-left"
Replace: data-anim="fadeInLeft"

Find:    data-aos="fade-right"
Replace: data-anim="fadeInRight"

Find:    data-aos="fade"
Replace: data-anim="fadeIn"

Find:    data-aos="zoom-in"
Replace: data-anim="zoomIn"

Find:    data-aos-
Replace: data-anim-
```

## What You Gain

After migrating, you get these features for free:

- **Anti-FOUC protection** — No more flash of content on page load
- **Smaller bundle** — From ~14KB to under 2KB
- **More triggers** — hover, click, and focus triggers (AOS only supports viewport)
- **Stagger patterns** — center, edges, start, end patterns
- **No init() call** — Remove your `AOS.init()` and JS setup entirely
- **Responsive controls** — Disable or swap animations per breakpoint
