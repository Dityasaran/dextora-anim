# Responsive

Not every animation makes sense on every screen size. data-anim provides responsive attributes to disable or swap animations based on viewport width.

## Disable Animations

Use `data-anim-disable` to turn off animations at specific breakpoints.

<AttributeTable attributes={[
  { name: 'data-anim-disable', type: '"mobile" | "tablet" | "desktop"', description: 'Disable the animation at the specified breakpoint.' },
]} />

<CodeBlock
  code={'<!-- No animation on mobile -->\n<div\n  data-anim="slideInLeft"\n  data-anim-disable="mobile"\n>\n  This only animates on tablet and desktop\n</div>\n\n<!-- No animation on tablet -->\n<div\n  data-anim="fadeInUp"\n  data-anim-disable="tablet"\n>\n  Skips animation on tablet\n</div>'}
  lang="html"
/>

### Breakpoints

| Breakpoint | Range |
|-----------|-------|
| `mobile` | < 768px |
| `tablet` | 768px - 1023px |
| `desktop` | >= 1024px |

## Mobile Override

Use `data-anim-mobile` to swap to a different (typically simpler) animation on mobile.

<AttributeTable attributes={[
  { name: 'data-anim-mobile', type: 'string', description: 'Alternative animation name for mobile screens (< 768px).' },
]} />

<CodeBlock
  code={'<!-- Full slide on desktop, simple fade on mobile -->\n<div\n  data-anim="slideInLeft"\n  data-anim-mobile="fadeIn"\n>\n  Responsive animation\n</div>'}
  lang="html"
/>

This is useful when:
- Complex animations cause jank on low-end mobile devices
- Horizontal slide animations don't work well on narrow screens
- You want a more subtle experience on mobile

## Best Practices

### Prefer simpler animations on mobile

<CodeBlock
  code={'<!-- Good: simpler animation on mobile -->\n<div data-anim="slideInLeft" data-anim-mobile="fadeIn">\n  Content\n</div>\n\n<!-- Good: disable unnecessary animation -->\n<div data-anim="rotateIn" data-anim-disable="mobile">\n  Content\n</div>'}
  lang="html"
/>

### Consider reduced motion

data-anim automatically respects the `prefers-reduced-motion` media query. When the user has requested reduced motion, all animations are disabled system-wide. You don't need to handle this manually.

### Performance tips

| Animation type | Mobile performance | Recommendation |
|---|---|---|
| `fadeIn`, `fadeInUp` | Excellent | Use freely |
| `zoomIn`, `zoomOut` | Good | GPU-friendly, works well |
| `slideInLeft/Right` | Caution | Can cause horizontal overflow |
| `rotateIn`, `flip` | Caution | Use `data-anim-mobile="fadeIn"` |
| `bounce`, `wobble` | Good | Multi-step but GPU-friendly |
